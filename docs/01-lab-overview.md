# Lab overview

## Project name

IAM Access Governance Lab | Microsoft Entra ID

## Purpose

This project is a hands-on IAM access governance lab built in Microsoft Entra ID.

The goal is to practice common identity and access management tasks that an entry-level IAM Analyst may support, including user provisioning, role-based access assignment, MFA enforcement, Conditional Access, privileged access management, joiner/mover/leaver workflows, and access review simulation.

## Lab environment

The lab uses a Microsoft Entra ID tenant named **Northstar Identity Lab**.

The environment includes fictional users, departments, and security groups to model realistic identity governance scenarios across HR, Finance, IT, Conditional Access, privileged access, and access review workflows.

## Scope

This lab includes:

- Microsoft Entra tenant setup
- Test users
- Security groups
- Role-based access assignments
- MFA and Conditional Access
- Joiner, mover, and leaver workflows
- Privileged Identity Management
- Access review simulation
- IAM documentation and screenshot-based evidence

## Out of scope

This lab does not configure live integrations with Okta or SailPoint.

Okta and SailPoint may be referenced later as conceptual comparisons only.
